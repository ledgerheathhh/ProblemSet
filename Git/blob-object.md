# Git 原理解析：Blob 对象

在 Git 的内部存储模型中，一切内容都以 **对象（Object）** 的形式保存。Git 共有四种核心对象：

* **blob**：存储文件内容
* **tree**：存储目录结构
* **commit**：存储提交信息
* **tag**：存储标签

其中 **blob（Binary Large Object）** 是最基础的一种对象，它只负责 **保存文件内容本身**。

---

# 一、Blob 对象是什么

Blob 可以理解为：

> Git 中保存文件内容的一种数据对象。

Blob **只存储文件内容，不存储任何文件名、路径、权限等信息**。

例如：

```
hello.txt
内容: Hello Git
```

Git 实际存储的是：

```
Hello Git
```

而 **hello.txt 这个名字是由 tree 对象记录的**。

---

# 二、Blob 的生成过程

当你执行：

```bash
git add hello.txt
```

Git 会做以下事情：

1. 读取文件内容
2. 生成对象头
3. 计算 SHA-1 哈希
4. 压缩存储到 `.git/objects`

### 对象结构

Git 在计算 hash 前，会构造：

```
blob <文件字节数>\0<文件内容>
```

例如：

```
blob 9\0Hello Git
```

然后对这段数据计算 **SHA-1**：

```
e965047ad7c57865823c7d992b1d046ea66edf78
```

---

# 三、Blob 存储结构

Git 将对象存储在：

```
.git/objects
```

结构如下：

```
.git
 └── objects
      ├── e9
      │   └── 65047ad7c57865823c7d992b1d046ea66edf78
```

规则：

```
前2位作为目录
剩余38位作为文件名
```

示例：

```
e9/65047ad7c57865823c7d992b1d046ea66edf78
```

# 五、Blob 与文件关系

Blob **与文件名没有关系**。

例如：

```
hello.txt   -> 内容: Hello
hi.txt      -> 内容: Hello
```

Git 只会生成 **一个 blob**：

```
blob: "Hello"
```

因为 **内容相同，hash 相同**。

对应关系如下：

```
           +---------+
hello.txt ->|        |
hi.txt    ->| blob   | -> "Hello"
           |        |
           +---------+
```

这也是 Git **节省空间的重要原因**。

---

# 六、Blob、Tree、Commit 的关系

Git 的对象关系结构：

```
        commit
           |
           v
         tree
        /    \
       v      v
    blob    blob
   file1    file2
```

结构说明：

```
commit
  └── tree (目录)
        ├── blob (file1内容)
        └── blob (file2内容)
```

也就是说：

```
commit 记录一次提交
tree   记录目录结构
blob   记录文件内容
```

---

# 七、查看 Blob 对象

可以通过命令查看：

### 查看对象类型

```bash
git cat-file -t <hash>
```

输出：

```
blob
```

### 查看对象内容

```bash
git cat-file -p <hash>
```

输出：

```
Hello Git
```

---

# 八、总结

Blob 的核心特点：

| 特点          | 说明        |
| ----------- | --------- |
| 只存储内容       | 不包含文件名    |
| 通过 SHA-1 标识 | 内容决定 Hash |
| 自动去重        | 相同内容只存一份  |
| zlib 压缩     | 节省存储空间    |

一句话总结：

> **Blob 是 Git 中用于存储文件内容的不可变对象，内容相同的文件会共享同一个 Blob。**
